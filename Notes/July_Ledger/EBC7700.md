# RISC-V GCC/LLVM 工具链测试步骤：EBC7700

## 1. 测试用例及说明

测试 EBC7700 开发板上通用 RISC-V GCC 和 LLVM 工具链的支持情况，验证 GCC 和 LLVM 工具链是否能在目标 RISC-V Linux 环境中正常安装、编译和运行程序。通过 Hello World 和 CoreMark 两个程序验证基本功能和性能。使用 RuyiSDK 作为统一的工具链管理工具，确保测试环境的一致性。

本测试不以开发板厂商特定 IDE、厂商 SDK 工具链、ESSDK 或 ESWIN 版 FFmpeg 作为测试前提。上述工具只在需要验证板载 NPU、DSP、硬件音视频等厂商能力时另行测试；本用例仅覆盖通用 RISC-V Linux C/C++ 编译运行能力。

测试流程为：

- 系统初始化：系统镜像下载与烧录，串口连接与登录，网络配置。
- 开发板信息确认：确认系统版本、内核版本、架构、CPU 核心、ISA、uarch。
- 工具链方案确认：确认本次使用 Ruyi 管理的通用 `gnu-plct` 和 `llvm-plct` 工具链。
- 扩展参数确认：根据板端 ISA 输出整理用户态可用扩展，并在执行 CoreMark 前确认最终 `-march/-mabi`。
- 环境准备：安装系统依赖包，安装 Ruyi 包管理器，通过 Ruyi 包管理器安装 `gnu-plct` 和 `llvm-plct` 工具链。
- GCC 测试：创建并激活 GCC 虚拟环境，验证 GCC 版本，编译并运行 Hello World，编译并运行 CoreMark 基准测试。
- LLVM 测试：创建并激活 LLVM 虚拟环境，验证 Clang 版本，编译并运行 Hello World，编译并运行 CoreMark 基准测试。
- 环境清理：退出虚拟环境，完成测试。

开发板型号列表如下：

| 序号 | 开发板 | 处理器 | IP 核（主 CPU） |
|---|---|---|---|
| 1 | EBC7700 / EBC77 Series SBC | EIC7700X / EIC770x | ESWIN EIC770x，4 核 RV64 |

## 2. EBC7700 扩展参数依据

根据板端实际输出文件，EBC7700 系统和 CPU 信息如下：

```text
Linux ubuntu 6.6.115-2025-eic7700 #12.003 SMP Mon Mar 9 15:50:21 UTC 2026 riscv64 GNU/Linux
Ubuntu 24.04.4 LTS
isa: rv64imafdch_zicntr_zicsr_zifencei_zihpm_zba_zbb_sscofpmf
mmu: sv48
uarch: eswin,eic770x
```

CoreMark 编译时尽可能打开 EBC7700 支持的用户态可用扩展。由于 `sscofpmf` 属于 supervisor-level 扩展，普通用户态 Hello World/CoreMark 不建议放入 `-march`。因此本测试使用如下参数：

```text
-march=rv64imafdch_zicntr_zicsr_zifencei_zihpm_zba_zbb -mabi=lp64d
```

本测试默认在 EBC7700 板端原生编译并运行，目标 Ubuntu 系统提供本机动态运行环境，因此 CoreMark 命令默认不添加 `-static`。如动态链接运行失败，可在 `XCFLAGS` 中追加 `-static` 后重新编译，并在测试记录中说明。

如当前 Ruyi 工具链不接受完整扩展串，可按如下顺序回退并记录原因：

```text
rv64imafdch_zicntr_zicsr_zifencei_zihpm_zba_zbb
rv64gc_zba_zbb
rv64gc
```

Hello World 仅用于验证基本编译运行闭环，不强制打开全部扩展；CoreMark 承担“打开所有能开的用户态扩展”的验证。执行 CoreMark 前，需要先确认最终使用的扩展参数：

```text
确认项：是否使用 -march=rv64imafdch_zicntr_zicsr_zifencei_zihpm_zba_zbb -mabi=lp64d
确认结论：是 / 否
确认人：
备注：
```
