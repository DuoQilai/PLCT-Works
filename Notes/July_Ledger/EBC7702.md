# RISC-V GCC/LLVM 工具链测试步骤：EBC7702

## 1. 测试用例及说明

测试 EBC7702 / EBC77 Series EBC7702 Mini-DTX Mainboard 上通用 RISC-V GCC 和 LLVM 工具链的支持情况，验证 GCC 和 LLVM 工具链是否能在目标 RISC-V Linux 环境中正常安装、编译和运行程序。通过 Hello World 和 CoreMark 两个程序验证基本功能和性能。使用 RuyiSDK 作为统一的工具链管理工具，确保测试环境的一致性。

本测试不以开发板厂商特定 IDE、ESSDK、ESWIN 版 FFmpeg、NPU/视频/音频专用 SDK 作为测试前提。上述工具只在需要验证板载 NPU、DSP、硬件音视频等厂商能力时另行测试；本用例仅覆盖通用 RISC-V Linux C/C++ 编译运行能力。

测试流程为：

- 系统初始化：系统镜像下载与烧录，串口连接与登录，网络配置。
- 开发板信息确认：确认系统版本、内核版本、架构、CPU 核心、ISA、uarch。
- 工具链方案确认：确认本次使用 Ruyi 管理的通用 `gnu-plct` 和 `llvm-plct` 工具链。
- 扩展参数确认：根据板端 ISA 输出整理用户态可用扩展，并在执行 CoreMark 前确认最终 `-march/-mabi`。
- 环境准备：安装系统依赖包，安装 Ruyi 包管理器，通过 Ruyi 包管理器安装 `gnu-plct` 和 `llvm-plct` 工具链。
- GCC 测试：创建并激活 GCC 虚拟环境，验证 GCC 版本，编译并运行 Hello World，编译并运行 CoreMark 基准测试。
- LLVM 测试：创建并激活 LLVM 虚拟环境，验证 Clang 版本，编译并运行 Hello World，编译并运行 CoreMark 基准测试。
- 环境清理：退出虚拟环境，完成测试。

开发板型号列表如下：

| 序号 | 开发板 | 处理器 | IP 核（主 CPU） |
|---|---|---|---|
| 1 | EBC7702 / EBC77 Series EBC7702 Mini-DTX Mainboard | EIC7702X | ESWIN EIC7702X，8 核 RV64 |

烧录前，将 bootloader 和解压后的 Ubuntu 镜像放到 ext4 格式 U 盘：

```text
USB/
  bootloader_EBC7702-D01_die0.bin
  bootloader_EBC7702-D01_die1.bin
  d560-24.04-preinstalled-server-riscv64_20260313_1628_70.img
```

进入 U-Boot 后，如需烧录或更新 bootloader：

```text
usb reset
ls usb 0
ext4load usb 0 0x100000000 bootloader_EBC7702-D01_die0.bin
es_burn write 0x100000000 flash
ext4load usb 0 0x100000000 bootloader_EBC7702-D01_die1.bin
es_burn write 0x100000000 flash 1
```

如果板上 bootloader 已经兼容，可跳过 bootloader 烧录。

烧录 Ubuntu 镜像到 `mmc 0`：

```text
es_fs write usb 0 d560-24.04-preinstalled-server-riscv64_20260313_1628_70.img mmc 0
reset
```

## 6. ESSDK / FFmpeg 说明

官方 README 中的 `Essdk deb packages install` 不是本测试的前置步骤，而是系统启动后安装 ESWIN SDK 和 FFmpeg deb 包。

含义：

- `ESSDK`：ESWIN 提供的硬件能力 SDK，用于 NPU、DSP、音视频、内存管理等模块。
- `FFmpeg`：音视频工具，这里的 FFmpeg 可能集成了 ESWIN 的硬件编解码插件。

离线安装方式示例：

```bash
unzip 251230_essdk_ffmpeg.zip
sudo dpkg -i *.deb
sudo apt -f install
```

它不是 Hello World/CoreMark 测试所需工具，也不是开发板唯一开发方式。只有调用板载 NPU、DSP、硬件音视频等能力时，通常需要 ESSDK 或 ESWIN 版 FFmpeg。
