# RISC-V GCC/LLVM 工具链测试步骤：HiFive Premier P550

## 1. 测试用例及说明

测试 HiFive Premier P550 开发板上 GCC 和 LLVM 工具链的支持情况，验证 GCC 和 LLVM 工具链是否能在目标 RISC-V 开发板上正常安装、编译和运行程序。通过 Hello World 和 CoreMark 两个程序验证基本功能和性能。使用 RuyiSDK 作为统一的工具链管理工具，确保测试环境的一致性。

测试流程为：

- 系统初始化：硬件连接、启动模式确认、串口连接、系统启动和登录。
- 环境准备：确认系统信息，安装系统依赖包，安装 Ruyi 包管理器，通过 Ruyi 包管理器安装 `gnu-plct` 和 `llvm-plct` 工具链。
- GCC 测试：创建并激活 GCC 虚拟环境，验证 GCC 版本，编译并运行 Hello World，编译并运行 CoreMark 基准测试。
- LLVM 测试：创建并激活 LLVM 虚拟环境，验证 Clang 版本，编译并运行 Hello World，编译并运行 CoreMark 基准测试。
- 环境清理：退出虚拟环境，归档测试证据，完成测试。

开发板型号列表如下：

| 序号 | 开发板 | 处理器 | IP 核（主 CPU） |
|---|---|---|---|
| 1 | HiFive Premier P550 / HF106 | ESWIN EIC7700X | SiFive P550，4 核 RV64 |

## 2. P550 扩展参数依据

根据 HiFive Premier P550 入门指南，HF106 使用 ESWIN EIC7700X SoC 和 SiFive 四核 P550，板载 16GB/32GB LPDDR5、128GB eMMC，并可预装 SiFive Freedom-U-SDK 或 Ubuntu Linux Distribution。本文默认按 64 位 Linux 开发板在板端原生编译并运行测试。

当前 P550 板端实测信息系统和 CPU 信息如下：

```text
Linux ubuntu 6.6.92-2025-eic7700 #07 SMP Mon Sep 1 04:17:25 UTC 2025 riscv64 GNU/Linux
Ubuntu 24.04.3 LTS
isa: rv64imafdch_zicntr_zicsr_zifencei_zihpm_zba_zbb_sscofpmf
mmu: sv48
uarch: eswin,eic770x
```

测试前必须在板端记录以下信息，作为参数依据和测试环境证据：

```bash
uname -a
uname -m
cat /etc/os-release
cat /proc/cpuinfo | head -80
lscpu
```

CoreMark 编译时尽可能打开 P550 支持的用户态可用扩展。由于 `sscofpmf` 属于 supervisor-level 扩展，普通用户态 Hello World/CoreMark 不建议放入 `-march`。根据板端 ISA，可先验证如下完整用户态扩展参数：

```text
-march=rv64imafdch_zicntr_zicsr_zifencei_zihpm_zba_zbb -mabi=lp64d
```

其中 `rv64imafdch` 包含 64 位整数、乘除、原子、单/双精度浮点、压缩和 hypervisor 扩展；`zicntr`、`zicsr`、`zifencei`、`zihpm` 为计数器、CSR、取指栅栏和硬件性能计数器相关扩展；`zba`、`zbb` 为位操作扩展子集。若当前工具链不接受 `h` 或某个 Z 扩展，应记录失败原因并按回退顺序测试。

当前 GCC CoreMark 实测通过记录见 [2p550.txt](./2p550.txt)，实际通过参数为：

```text
-march=rv64gc_zba_zbb -mabi=lp64d
```

如当前 Ruyi 工具链不接受完整扩展串，或本轮只按已验证参数复现结果，可按如下顺序回退并记录原因：

```text
rv64gc_zba_zbb
rv64gc
```

Hello World 仅用于验证基本编译运行闭环，不强制打开全部扩展；CoreMark 承担“打开所有能开的扩展”的验证。
